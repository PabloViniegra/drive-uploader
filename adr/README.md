# Architectural Decision Records

This folder records architecturally significant decisions for `drive-uploader`. Each ADR captures the context, the decision, the alternatives considered with their scores, the consequences (positive and accepted negatives), and the implementation notes that are still in motion.

When a decision supersedes a prior one, the older ADR's status is moved to `Superseded by ADR-NNN` and the new ADR records `**Supersedes:** ADR-MMM` in its frontmatter.
